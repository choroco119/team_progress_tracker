# 部品手配の未着手アラートの修正

部品手配の3つのカテゴリ（主・予・支）のうち、一部が「不要」に設定されている場合でも、残りの「要」設定カテゴリが未着手であれば正しくアラートアイコンが表示されるように修正します。

## ユーザーレビューが必要な項目
- なし

## 変更内容

### [JS] [app.js](file:///c:/Users/kohei/.gemini/antigravity/scratch/team_progress_tracker/app.js)

1.  **判定ロジックの改善**:
    - `partsSummary` の内容（表示用テキスト）とは別に、`hasUnstartedRequired`（「要」設定だが未着手の項目があるか）というフラグを導入します。
2.  **表示ロジックの修正**:
    - `hasUnstartedRequired` が真であり、かつ納入仕様書が着手済（`isSpecStarted`）の場合、他の「不要」メッセージの有無にかかわらずアラートアイコンを表示するようにします。

## 検証計画
### 手動検証
1.  納入仕様書の「出図」を「済」にする。
2.  部品手配の要否設定で「主部品」を「要」、「予備品」を「否」にする。
3.  主部品が未着手の状態で、一覧表に「要着手」のアラートアイコンと「不要(予)」の両方が表示されることを確認する。
4.  主部品を手配済みにすると、アラートアイコンが消えることを確認する。
