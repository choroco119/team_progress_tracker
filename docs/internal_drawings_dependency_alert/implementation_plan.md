# 社内工事図の着手アラート

社内工事図の着手に必要な前工程（板金詳細図送付、部品手配、銘板手配）が全て完了した際、社内工事図が未着手であればアラートアイコンを表示します。

## ユーザーレビューが必要な項目
- なし

## 変更内容

### [JS] [app.js](file:///c:/Users/kohei/.gemini/antigravity/scratch/team_progress_tracker/app.js)

1.  **前工程の完了判定**: `renderTable` 関数内で、以下の条件が全て満たされているか判定します (`isReadyForInternalDrawings`)。
    - 板金手配: 不要、または詳細図送付済。
    - 部品手配: 全ての要設定カテゴリが手配依頼済。
    - 銘板手配: 不要、または注文書送付済。
2.  **社内工事図のアラート表示**:
    - `nec.internalDrawings` が「要」かつ `issueDate` が未入力、かつ前工程が全て完了している場合、「要着手」のアラートアイコンを表示します。

## 検証計画
### 手動検証
1.  板金手配、部品手配、銘板手配の「要」となっているものを全て完了させる。
2.  社内工事図が未着手（出図前）の案件に `！要着手` アラートアイコンが表示されることを確認。
3.  前工程が一つでも未完了（例：板金の詳細図がまだ）の場合は、アラートが表示されないことを確認。
4.  社内工事図を出図済みにすると、アラートが消えることを確認。
