# 検索機能付きコピー機能（高度な検索）の実装計画 (Searchable Copy Plan)

製番が増加しても、目的の案件を「製番・客先・件名」などのキーワードですぐに見つけ出せる検索UIを実装します。

## 変更内容

### 1. UIの刷新 (`index.html`)
- 現在の `<select>` 形式を廃止し、以下の構成に変更。
    - **検索入力欄 (`input`)**: キーワード入力用。
    - **検索結果リスト (`ul/div`)**: 絞り込まれた候補を表示するフローティングリスト。
- **モーダル構造の最適化**: 検索結果がモーダルの外側まで表示されるよう、CSS の `overflow` 設定を調整し、HTML のネスト構造を修復。

### 2. 検索・絞り込みロジックの実装 (`app.js`)
- `updateCopySearch(query)`: 入力された文字列に基づき、`state.projects` から「製番」「客先」「件名」を対象に部分一致検索を実行。
- **検索結果の表示**: 最大15件の候補をリスト表示。ホバー時の視認性向上。
- **堅牢性の向上**: `toggleModal` ヘルパー関数の導入により、DOM要素の状態に関わらず安全にモーダルを制御。

### 3. コピー適用処理の統合
- リスト内の項目がクリックされたら、その案件のデータを取得してフォームにセット（工程要否も含む）し、検索リストを閉じる。

### 4. 表レイアウトの調整
- 「操作」列を削除し、各セル（IDや担当者など）のクリックで編集・詳細モーダルを開くよう統合。

## メリット
- 製番を正確に覚えていなくても、客先名などで検索してコピーできる。
- 数百件、数千件のデータがあっても、数文字の入力で目的の案件に到達できる。
- UIがよりクリーンになり、データ表示領域が拡大。

## 検証計画
- 検索窓に製番の一部を入力し、候補が出ることを確認。
- 候補リストがモーダルの枠に蹴られず表示されることを確認。
- 工程セルをクリックした際、詳細モーダルが干渉なく開くことを確認。
- データのコピー後、新規登録が正常に完了することを確認。
