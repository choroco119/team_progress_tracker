# 強制上書きの禁止と競合回避の厳格化計画 (Strict Conflict Prevention Plan)

データ整合性を最優先するため、編集中に他人がデータを更新した際、自分の内容で強制的に上書きすることを禁止します。

## 変更内容

### 1. 競合検知時の挙動変更 (`app.js`)
- `saveWithSync` 関数内での `confirm` ダイアログを廃止。
- 代わりに `alert` を表示し、「他人が更新したため、最新データを読み込む」旨を通知。
- 通知後、自動的に `loadFromFolder` を実行して、保存処理を中断する。

## メリット
- 他人の最新の編集内容を誤って古いデータで上書きしてしまう事故を100%防止できる。

## 注意点（ユーザーへの影響）
- 同時に同じ案件を編集していた場合、後から保存しようとした人の入力内容は破棄され、最新データに置き換わります。
- 衝突を避けるため、各工程モーダルを開く直前の `refreshIfRemoteUpdated`（最新チェック）がより重要になります。

## 検証計画
- 2つのブラウザを開き、片方で保存。もう片方で直後に保存を試みた際、上書きの選択肢が出ず、強制的に最新データに切り替わることを確認。
