# 工程間の繋がりによる着手アラート（アイコン表示）

納入仕様書のいずれかの工程が完了している（出図、返却、または承認のいずれかが済んでいる）場合に、板金手配や部品手配が「要」設定であるにもかかわらず未着手である案件に対して、注意を促すアラートアイコン（！）を表示します。

## ユーザーレビューが必要な項目
- なし

## 変更内容

### [JS] [app.js](file:///c:/Users/kohei/.gemini/antigravity/scratch/team_progress_tracker/app.js)

1.  **工程着手状況の判定**: `renderTable` 関数内で、納入仕様書が「着手可能状態」（いずれかの工程が済）であるかを判定します。
2.  **板金手配のアラート表示**:
    - `nec.sheetMetal` が「要」かつ `quoteDate` が未入力、かつ納入仕様書が着手済の場合、セルにアラートアイコンを表示します。
3.  **部品手配のアラート表示**:
    - 部品手配のいずれかのカテゴリ（主、予、支）が「要」かつ、全ての要設定カテゴリにおいて `requestDate` が未入力、かつ納入仕様書が着手済の場合、セルにアラートアイコンを表示します。

### 使用するアイコン
- Lucideの `alert-circle` アイコン（色は `var(--warning-color)`）を使用します。

## 検証計画
### 手動検証
1.  納入仕様書の「出図」を「済」にする。
2.  板金手配が「要」で、まだ「見積依頼送付」などがされていない案件にアラートアイコンが表示されることを確認。
3.  部品手配が「要」で、まだどのカテゴリも「手配依頼」されていない案件にアラートアイコンが表示されることを確認。
4.  板金手配または部品手配のいずれかのアクション（見積依頼、手配依頼など）を行うと、そのセルのアイコンが消えることを確認。
5.  納入仕様書が全て未着手（-表示）の場合は、アラートアイコンが表示されないことを確認。
