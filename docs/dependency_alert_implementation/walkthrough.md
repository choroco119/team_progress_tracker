# 変更内容のまとめ (Walkthrough) - 工程間連携アラート

納入仕様書の進捗に合わせ、着手可能になった板金手配・部品手配の放置を防止するためのアラート機能を実装しました。

## 1. アラートの発生条件
以下の全ての条件を満たす場合にアラートが表示されます。
- **前工程の進捗**: 納入仕様書で「出図」「返却」「承認」のいずれかが行われている。
- **自工程の要否**: 板金手配または部品手配が「要」に設定されている。
- **未着手状態**: 
    - 板金手配: 見積依頼がまだ送付されていない。
    - 部品手配: 全ての必要なカテゴリ（主・予・支）で手配依頼がまだされていない。

## 2. UIの表示
対象のセルに、オレンジ色のアラートアイコン（！）と「要着手」というテキストを表示します。

| 工程 | 通常時 | アラート表示時 |
| :--- | :--- | :--- |
| **板金手配** | - | <i data-lucide="alert-circle" style="color:var(--warning-color);"></i> **要着手** |
| **部品手配** | - | <i data-lucide="alert-circle" style="color:var(--warning-color);"></i> **要着手** |

## 3. 検証内容
- 納入仕様書を出図済みにすると、未着手の板金・部品手配にアイコンが表示されることを確認。
- 板金手配で見積依頼を入力すると、アイコンが消えて「見積依頼送付済」に変わることを確認。
- 部品手配でいずれか一つのカテゴリを手配依頼すると、アイコンが消えてそのカテゴリの進捗が表示されることを確認。
- 納入仕様書が未着手の場合は、アラートが表示されないことを確認。
