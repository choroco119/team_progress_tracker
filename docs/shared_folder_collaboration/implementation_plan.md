# 共有フォルダを利用した多人数同期・排他制御の実装計画

共有フォルダ（LAN内の共有ドライブ等）にデータを配置し、多人数で利用する際のデータ衝突を防ぐ仕組みを実装します。

## 課題
現在の実装では、個別のJSONファイルを「読み込み・保存」する形式のため、複数のユーザーが同時に編集して保存すると、後に保存した人が前の人の変更を上書きして消してしまう問題があります。

## 解決策
ブラウザの `window.showDirectoryPicker()`（ディレクトリ選択）を活用し、以下の機能を実装します：
1. **ロックファイル制御**: 保存中に `lock.json` を作成し、他人の保存を一時的にブロックします。
2. **競合検知（楽観的ロック）**: データに最終更新日時（`lastUpdated`）を持たせ、自分が読み込んだ後に他人が更新していないか保存直前にチェックします。
3. **自動リフレッシュ**: フォルダ内のファイルの更新を定期的に監視し、最新データがある場合に通知します。

## User Review Required

> [!IMPORTANT]
> **共有フォルダの選択**: 初回利用時、またはブラウザを再起動した際に、ユーザーが一度だけ「データ保存先のフォルダ」を選択する必要があります（ブラウザのセキュリティ制限のため）。

> [!WARNING]
> **ネットワークの切断**: 共有フォルダ（LAN）への接続が切れた状態で保存しようとするとエラーが発生します。その場合、データをローカルにバックアップとしてダウンロードする救済措置を設けます。
