# 共有フォルダ同期・排他制御の実装完了

外部サービスを使わず、LAN内の共有フォルダを介して多人数で安全にデータを共有できる仕組みを実装しました。

## 実装内容

### 1. フォルダ同期 UI
ヘッダーに以下の要素を追加しました：
- **同期インジケーター**: 現在の接続状態（未接続/同期中/処理中）と、最終同期時刻を表示します。
- **「保存同期」ボタン**: 編集内容を共有フォルダに書き込みます。この際、後述するロックと競合チェックが行われます。
- **「同期設定」ボタン**: 初回利用時や、保存先フォルダを変更したい場合に使用します。

### 2. 排他制御 (Locking)
保存ボタンを押すと、フォルダ内に `lock.json` を作成します。
- 他のユーザーが同時に保存しようとすると、「他の人が保存中です」と警告が出て上書きを防止します。
- 万が一、ブラウザがクラッシュしてロックファイルが残ってしまった場合でも、5分以上経過していれば自動的に解除される安全策を講じています。

### 3. 楽観的ロック (Conflict Detection)
自分がファイルを読み込んだ後に、他の誰かが保存を完了させていた場合、保存時に以下の警告が出ます：
> 「編集中に他の人がデータを更新しました。強制的に上書きしますか？」
- 「キャンセル」を選ぶと、最新データを読み込み、自分の編集内容は破棄されます（上書きによるデータ消失を防止）。
- 「OK」を選ぶと、自分の内容で上書きします。

### 4. 自動更新 (Auto Refresh)
30秒ごとに共有フォルダ内のファイルをチェックし、更新があれば自動的に最新の状態を画面に反映します。

## 検証結果
- **同一フォルダ接続**: 正常に動作。
- **同時保存時のロック**: 正常に動作し、後からの保存をブロック。
- **競合検知**: 正常に動作し、上書き警告を確認。
- **自動更新**: 30秒以内に他人の変更が反映されることを確認。

## 使用方法
1. 画面右上の「同期設定」をクリックし、社内共有フォルダ内の適切な場所（または新規作成したフォルダ）を選択してください。
2. 以降は「保存同期」ボタンを押すだけで、安全にデータが共有されます。
3. ブラウザを再起動した際は、再度「同期設定」からフォルダを選択する必要があります。
