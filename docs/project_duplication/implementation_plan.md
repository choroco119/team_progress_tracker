# 製番コピー登録機能の実装計画 (Project Duplication Plan)

既存の製番情報を流用して、新しい製番を効率的に登録できる「コピー登録機能」を実装します。

## 変更内容

### 1. UIの変更
- **`index.html`**: テーブルヘッダーの「納期」の後に「操作」列を追加。
- **`app.js` (`renderHeader`)**: JS側でも「操作」列のヘッダー管理を同期。
- **`app.js` (`renderTable`)**: 
    - 各行にコピーボタン（`copy` アイコン）を追加。
    - ボタンのツールチップに「コピーして新規作成」と表示。

### 2. コピー処理の実装 (`app.js`)
- `openCopyModal(project)` 関数を追加。
- 指定されたプロジェクトの情報をフォームにセット。
- **引き継ぐ**: 客先、件名、納期、担当者、工程の要否設定（necessity）など。
- **引き継がない**: 製番（ID）、各工程の進捗状況（日付、ステータス）。
- `editingProjectId` を `null` に設定し、登録時は `addNewProject` が呼ばれるようにする。

## 検証計画
- 既存案件の「コピー」ボタンを押し、製番以外の情報が正しく入力されていることを確認。
- 工程の「要否」設定が引き継がれていることを確認。
- 新しい製番を入力して登録し、進捗状況（日付等）が空の状態で新規作成されることを確認。
- 共有フォルダへの自動同期が正しく行われることを確認。
