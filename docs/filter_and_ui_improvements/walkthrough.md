# 変更内容のまとめ (Walkthrough)

本セッションでは、Engineering Progress Tracker のフィルタ機能の大幅な強化と、ユーザー体験を向上させるUIの微調整を行いました。

## 1. 詳細工程フィルタの実装
各工程において、より実務に即した状態での絞り込みが可能になりました。

| 工程 | フィルタ内容 |
| :--- | :--- |
| **納入仕様書** | 未出図、未返却、修正要、未承認 |
| **板金手配** | 見積依頼未送付、注文書未送付、詳細図未送付 |
| **部品手配** | 未手配（要設定がすべて済でない） |
| **銘板手配** | 未手配（注文書送付が未） |
| **社内工事図** | 未出図（出図が未） |
| **ソフト** | 未作成、未デバッグ |

## 2. 完了案件の表示制御の改善
- **デフォルト非表示**: 起動時は未完了の案件のみが表示されます。
- **「完了を表示」ボタン**: 必要な時だけ完了案件を一覧に追加できます。
- **フィルタ連動**: 工程フィルタ使用中は、完了済み案件は自動的に除外されます。
- **リセット保護**: 「全て解除」ボタンを押しても、完了の表示/非表示設定は維持されます。

## 3. 連動ハイライト機能
- 行のどこ（右側の工程セルなど）にマウスを乗せても、その案件の**左側（製番情報エリア）が常にハイライト**されます。
- 工程セル自体も個別にハイライトされ、クリック対象が明確になります。

## 4. ドキュメントの整理と一元化
- 過去の古い計画書や作業ログ（13フォルダ）を削除。
- 最新の仕様に基づいた3つの仕様書を `docs/system_spec/` に集約。
  - `functional_spec.md` (機能仕様)
  - `technical_spec.md` (技術仕様)
  - `ui_ux_guide.md` (UI/UXガイド)
- ルートの `README.md` を更新。

## 5. 検証内容
- 各工程フィルタを選択した際、意図した条件の案件のみが表示されることを確認。
- 工程が「不要」に設定されている案件が、フィルタ選択時に適切に除外されることを確認。
- 「完了を表示」ボタンのON/OFFで表示が切り替わり、工程フィルタ選択時に強制非表示になることを確認。
- ホバー時に製番情報エリアが連動してハイライトされることを確認。
