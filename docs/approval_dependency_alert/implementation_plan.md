# 納入仕様書の承認完了による後続工程アラート

納入仕様書が「承認済」になったことをトリガーに、板金手配（詳細図送付）と銘板手配の着手を促すアラートアイコンを表示します。

## ユーザーレビューが必要な項目
- なし

## 変更内容

### [JS] [app.js](file:///c:/Users/kohei/.gemini/antigravity/scratch/team_progress_tracker/app.js)

1.  **承認状況の判定**: `renderTable` 関数内で、納入仕様書が承認済（`spec.approvalDate` が存在）かを判定します (`isSpecApproved`)。
2.  **板金手配のアラート（詳細図送付）**:
    - `poSentDate` が済だが `drawingSentDate` が未入力、かつ納入仕様書が承認済の場合、「詳細図送付可」のアラートアイコンを表示します。
3.  **銘板手配のアラート**:
    - `nec.nameplateProcurement` が「要」かつ `poSentDate` が未入力、かつ納入仕様書が承認済の場合、「要着手」のアラートアイコンを表示します。

## 検証計画
### 手動検証
1.  納入仕様書の「承認」を「済」にする。
2.  銘板手配が「要」で未着手の案件にアラートアイコンが表示されることを確認。
3.  板金手配が「注文書送付済」で、詳細図が未送付の案件に「詳細図送付可」のアラートアイコンが表示されることを確認。
4.  それぞれの工程を進める（銘板の注文書送付、板金の詳細図送付）と、アラートが消えることを確認。
