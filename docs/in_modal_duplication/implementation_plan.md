# モーダル内コピー機能への移行計画 (In-Modal Duplication Plan)

一覧表の「操作」列を廃止し、新規登録モーダル内から既存情報をコピーできる、よりクリーンなUIへ移行します。

## 変更内容

### 1. 一覧表UIの復元
- **`index.html`**: 「操作」ヘッダーを削除。
- **`app.js` (`renderHeader`)**: `staticStartCount` を `8` に戻し、「操作」列の生成を削除。
- **`app.js` (`renderTable`)**: コピーボタン列の生成を削除。

### 2. モーダル内コピー機能の追加
- **`index.html`**: `add-project-form` の先頭に、既存製番の選択ドロップダウンと「コピー適用」ボタンを含むセクションを追加。
- **`app.js`**:
    - `updateCopyFromList()`: 既存の製番リストをドロップダウンに反映させる関数を追加。
    - `applyCopyFrom()`: 選択された製番の情報を現在のフォームに反映させる関数を追加。
    - **表示制御**: `editingProjectId` が `null`（新規登録時）の場合のみ、コピーセクションを表示するように調整。

## 検証計画
- 一覧表から「操作」列が消え、元のスッキリした見た目に戻っていることを確認。
- 新規登録モーダルを開いた際、上部にコピー用の選択メニューが表示されることを確認。
- 既存案件を選択して「コピー適用」を押すと、フォームに内容が正しくセットされることを確認。
- 既存案件の編集時は、コピー用メニューが表示されないことを確認。
