# 実装計画：同期処理ロジックの抽出（SyncManager）

## 目的
`app.js` に混在している「UI制御」と「ファイル同期ロジック」を分離し、メンテナンス性と堅牢性を向上させる。

## 実施内容
1. `js/sync-manager.js` の新規作成とロジックの移行
2. 同期状態変数の `js/utils.js` への集約
3. `app.js` からの同期呼び出しの置換
4. 差分比較テーブルのデザイン統一（追加要件）

## 完了定義
- 同期機能（読み書き、ロック、バックアップ）が正常に動作すること。
- `app.js` のコード量が削減され、可読性が向上すること。
- デザインの統一感が確保されていること。
