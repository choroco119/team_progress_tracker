# リファクタリング：差分比較ロジックの抽出

`app.js` に含まれていた進捗差分比較機能を、独立したエンジンとして `js/diff-engine.js` に抽出します。

## 目的
- `app.js` の責任範囲を「メイン画面の制御」に絞り、特定の機能（差分比較）を分離する。
- 差分比較に関連する状態（DOM要素、比較設定）を `DiffEngine` オブジェクト内にカプセル化する。
- コードの可読性とメンテナンス性を向上させる。

## 実施内容

### 1. `js/diff-engine.js` の作成
- `DiffEngine` オブジェクトを定義し、以下の機能を移植。
    - `init()`: DOM要素の取得とイベントリスナーの登録。
    - `updateBackupList()`: バックアップファイル一覧の取得。
    - `runComparison()`: バックアップとの比較実行。
    - `renderDiffResults()`: 結果のテーブル描画。
- グローバル変数（`window.dirHandle`, `window.state`）や共有ユーティリティ（`getFieldValue`, `showToast`）を利用して連携。

### 2. `index.html` の更新
- `js/utils.js` の後に `js/diff-engine.js` を読み込むように設定。

### 3. `app.js` のクリーンアップ
- 差分比較用のDOM定数（`diffBody` 等）を削除。
- 差分比較関連の関数群を削除。
- `init()` 内で `DiffEngine.init()` を呼び出すように変更。

## 検証プラン
- **タブ切り替え**: 「進捗差分比較」タブを開いた際にバックアップリストが正しく更新されるか。
- **比較実行**: バックアップを選択して比較を実行し、正しく結果が表示されるか。
- **UI整合性**: カラム幅や強調表示が以前の通り維持されているか。
