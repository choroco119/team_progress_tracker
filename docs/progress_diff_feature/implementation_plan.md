# 進捗差分比較機能の実装計画

過去のバックアップデータと現在のプロジェクトデータを比較し、指定された重要項目の進捗状況を一覧表示する機能を実装します。

## 目的
- 過去（昨日や一週間前など）の状態と現在の状態を比較し、どの案件が動いたかを即座に把握する。
- 納入仕様書、板金手配、主部品手配の重要項目に絞ることで、確認作業の効率を向上させる。

## 提案された変更

### UI/UXの設計
- **新しいタブの追加**: 「進捗一覧」に並ぶ形で「進捗差分比較」タブをメイン画面に追加。
- **カラム形式のレイアウト**: 
    - 1製番につき1行で表示。
    - 比較対象の9項目を個別の列（カラム）として配置し、横断的に確認可能にする。
    - 変更があったセルを太字＋成功色（緑）で強調表示。
- **ソート機能**: 納期の昇順で表示し、緊急度の高い案件から順に確認可能にする。

### ロジックの実装
- **バックアップ一覧の取得**: `File System Access API` を使用して `backups` フォルダ内の JSON ファイルを取得・表示。
- **データ比較エンジン**: 
    - 選択されたバックアップ JSON を読み込み、現在の `state.projects` とマッチング。
    - 以下の9項目に絞って比較を実行。
        - 納入仕様書: 出図 / 返却期日 / 承認
        - 板金手配: 手配先 / 納期(注文書) / 注文書受領 / 納期(詳細図) / 納期確定
        - 主部品手配: 納期
- **パフォーマンス最適化**: 全製番をループし、いずれかの項目に変更がある案件のみを抽出して描画。

## 検証プラン
- 異なる日付のバックアップを選択し、正しく変更箇所が強調されるか確認。
- 新規登録された案件が「新規登録」として正しく表示されるか確認。
- 1400px以上のテーブル幅が確保され、カラムが潰れずに表示されるか確認。
