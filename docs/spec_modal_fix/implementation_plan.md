# 納入仕様書モーダル 修正要否ボタンの状態反映の修正

納入仕様書モーダルにおいて、「修正要否（要・否）」ボタンを選択しても色が正しく切り替わらない問題を修正します。

## 現状の課題
- `app.js` 内で `classList.replace('btn-outline', 'btn-primary')` を使用している。
- `replace` メソッドは、置換対象のクラスが存在しない場合に何もしないため、初期状態や他の操作によってクラス構成が崩れると動作しなくなる。
- 納入仕様書以外の工程ボタン（「済」ボタンなど）でも同様の `replace` が使用されており、潜在的なバグとなっている可能性がある。

## 修正方針
1. `classList.replace` の使用を止め、`classList.add` と `classList.remove` を明示的に使用するように変更する。
2. ボタン群の状態を一括で制御するヘルパー関数を改善し、確実にクラスを適用する。
3. 納入仕様書モーダルだけでなく、他の工程モーダルでも同様の箇所があれば一括で修正し、UIの一貫性と動作の安定性を確保する。

## 予定される変更点

### app.js
- `renderSpecDocModal`: 修正要否ボタンのクラス操作ロジックの改善。
- `addSpecEventListeners`: 修正要否ボタンのクリックイベント内でのクラス操作ロジックの改善。
- その他、工程モーダルの `setBtnState` 的なロジックの見直し。

## 検証計画
### 手動検証
1. 納入仕様書モーダルを開き、「要」「否」を交互にクリックして色が正しく切り替わる（`btn-primary` が付与される）ことを確認。
2. 保存後に再度モーダルを開き、保存時の状態が正しく再現されていることを確認。
3. 他の工程（板金手配など）の「済」ボタンも正しく動作することを確認。
