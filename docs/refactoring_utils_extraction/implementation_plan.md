# リファクタリング：ユーティリティ関数の抽出

`app.js` の肥大化を解消し、メンテナンス性を向上させるための第一段階として、独立したユーティリティ関数を別ファイルに抽出します。

## 目的
- `app.js` のコード量を削減する（デグレのリスクを最小限に抑える）。
- 汎用的なロジック（数値フォーマット、日付判定など）を共通化し、再利用しやすくする。
- 密結合を避け、機能ごとのモジュール化への布石とする。

## 実施内容

### 1. `js/utils.js` の作成
以下の独立した関数を `app.js` から `js/utils.js` へ移動しました。
- `formatNumberWithCommas`: 数値のカンマ区切り
- `formatShortDate`: 日付の短縮表示
- `isOverdue`: 期限超過判定
- `isWithinTwoMonths`: 2ヶ月以内判定
- `showToast`: トースト通知表示
- `getFieldValue`: 工程データの取得
- `openDB`: IndexedDB接続（引数を受け取る汎用形式に変更）

### 2. `index.html` の更新
- `app.js` の読み込み前に `js/utils.js` を読み込むように設定。

### 3. `app.js` のクリーンアップ
- 抽出した関数の削除。
- `openDB` の呼び出し箇所を、引数を渡す新しい形式に更新。

## 検証プラン
- **表示確認**: 予算のカンマ表示や日付の表示が崩れていないか確認。
- **動作確認**: トースト通知が表示されるか確認。
- **同期確認**: IndexedDBを介したフォルダ記憶機能が正常に動作するか確認。
