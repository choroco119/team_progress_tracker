# 「部品手配」工程の追加計画

「部品手配」の新しい工程をシステムに追加します。この工程は「主部品手配」「予備品手配」「支給品手配」の3つのカテゴリで構成されます。

## > [!IMPORTANT] User Review Required
以下の仕様で実装を進めてよろしいか、内容をご確認ください。

### 1. データ構造
プロジェクトデータ (`state.projects[].processes.partsProcurement`) の中に以下を保存します。
*   `main` (主部品): `requestDate` (手配依頼日), `dueDate` (納期)
*   `spare` (予備品): `requestDate` (手配依頼日), `copyDate` (コピー), `dueDate` (納期)
*   `provided` (支給品): `requestDate` (手配依頼日), `dueDate` (納期)

### 2. モーダルのUIレイアウト
モーダル内は3つのセクションに見出しをつけて区切り、各項目を配置します。
「済」ボタンと日付入力の組み合わせは、既存の「納入仕様書」「板金手配」のルールを完全に踏襲します。

### 3. 一覧画面での表示（サマリーとツールチップ）
**一覧のセル（サマリー表示）:**
「主部品・予備品・支給品の納期の日付を表示する」とのご要望に基づき、一覧のセルには以下のように3つの納期をコンパクトに並べて表示します。

```text
主納期: 2026/5/10
予納期: 2026/5/15
支納期: - (未入力の場合)
```

**マウスオーバー時のポップアップ（ツールチップ）表示:**
セルにマウスを重ねた際は、他の工程と同様にすべての情報を表示します。

```text
[部品手配]
<主部品>
依頼: 2026/5/1
納期: 2026/5/10
<予備品>
依頼: 2026/5/2
コピー: 2026/5/3
納期: 2026/5/15
<支給品>
...
```
