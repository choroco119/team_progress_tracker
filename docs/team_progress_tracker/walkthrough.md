# Team Progress Tracker 開発の歩み (Walkthrough)

エンジニアリング進捗管理システムの初期開発フェーズが完了し、GitHubリポジトリへの初回プッシュまで完了しました。

## 実施した主な機能実装

### 1. 基盤レイアウトとデザイン
- **フルスクリーン対応**: ブラウザの高さに追従するモダンなレイアウト。
- **ダークモードUI**: 長時間の作業でも疲れにくい、洗練された配色。
- **Stickyヘッダー**: スクロールしても常にテーブルのヘッダーが表示される。
- **タブUI**: 「製番別進捗一覧」と将来の拡張用タブの切り替え。

### 2. 製番管理機能
- **詳細な案件登録**: 製番、納期、客先、担当者、件名、規格などの多彩な項目。
- **2カラム編集モーダル**: 左右に情報を整理し、視認性を高めた入力画面。
- **フォルダリンク連携**: ローカルパスやURLを登録し、ワンクリックでアクセス。

### 3. 工程管理（進捗トラッキング）
- **工程別の詳細入力**:
  - 納入仕様書、板金手配、部品手配（主・予・支）、銘板、社内図、ソフト。
  - 各工程ごとに専用のモーダルで日付やメモを管理。
- **進捗の可視化**: 一覧画面で「完了」「進行中」「不要」が色分けされ、一目で状況を把握。
- **詳細ツールチップ**: 行にマウスを重ねるだけで、全工程の詳細な日付やメモを確認。

### 4. 検索・絞り込み
- **マルチフィルタ**: 製番、客先、件名、担当者などによる自由入力検索。
- **工程未完フィルタ**: 「板金が未完了のものだけ表示」といった業務に直結した絞り込み。
- **完了案件の除外**: 現在進行中のプロジェクトに集中できるフィルタリング。

### 5. データ管理
- **マスターデータ設定**: 客先や担当者などをリストとして管理し、登録時にプルダウンで選択。
- **JSONファイル連携**: データベース不要で、読込・保存ボタンからファイルとしてデータを永続化。
- **Git/GitHub連携**: バージョン管理とコード管理の基盤を構築。
