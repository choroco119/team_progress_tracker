# 技術仕様書 (Technical Specification)

本アプリケーションのシステムアーキテクチャおよび技術的な実装詳細について記述します。

## 1. サーバーレス同期システム
ブラウザの **File System Access API** を使用し、社内共有フォルダ上の `data.json` を直接読み書きすることで、サーバー不要の共同編集を実現しています。

### 同期メカニズム
- **読み込み (`loadFromFolder`)**: 共有フォルダ内の `data.json` を取得し、グローバルな `state` 変数に展開。
- **書き込み (`saveWithSync`)**: `state` を JSON 化し、共有フォルダへ上書き保存。
- **自動更新**: 30秒ごとにバックグラウンドで `loadFromFolder` を実行し、他ユーザーの変更を検知。
- **アクション同期**: 「登録」「保存」「削除」の各アクション実行時に即座に同期を実行。

## 2. 厳格な排他制御と競合回避
データの整合性を守るため、以下の仕組みを導入しています。

- **ロックファイル (`data.lock`)**:
  - 書き込み開始前に `data.lock` を作成。
  - すでにロックファイルが存在する場合は、他のユーザーが編集中のため書き込みを中断。
  - 書き込み完了後にロックファイルを削除。
- **タイムスタンプによる競合検知**:
  - `data.json` 内の `lastUpdated` タイムスタンプを比較。
  - ローカルの最終同期時刻より共有フォルダのタイムスタンプが新しい場合、上書きを禁止し、最新データの読み込みを強制する。

## 3. 自動バックアップ
データ喪失リスクを最小限にするため、デイリーバックアップ機能を搭載しています。

- **保存場所**: 共有フォルダ内の `backups/` ディレクトリ。
- **トリガー**: 毎日、最初にアプリケーションに接続したユーザーが実行。
- **保持ポリシー**: 直近30日間分のスナップショットを保持。

## 4. 状態管理
アプリケーションの状態は、`app.js` 内の単一の `state` オブジェクトで管理されます。

```javascript
let state = {
    lastUpdated: "...", // 最終更新日時
    projects: [...],    // 案件データの配列
    config: {...}       // マスター設定（客先、スタッフ等）
};
```

## 5. UI/UX 実装
- **Lucide Icons**: アイコン表示に使用。
- **Vanilla CSS**: スタイル管理に使用。
- **Responsive Layout**: FHD解像度をターゲットとした効率的なデータ表示。
