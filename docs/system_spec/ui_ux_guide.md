# UI/UX ガイド (UI/UX Guide)

アプリケーションのデザイン原則とユーザー体験の設計指針について記述します。

## 1. デザイン原則
- **高密度な情報表示**: FHD（1920x1080）解像度で、できるだけ多くの案件をスクロールなしで確認できるレイアウト。
- **視認性の高い配色**:
  - **完了状態**: 完了済みのセルやボタンは薄緑色（`#4ade80`）で表示。
  - **不要工程**: 灰色のテキスト（`text-muted`）で表示し、ノイズを削減。
  - **警告表示**: 納期が近い、または過ぎている場合は赤（`#f87171`）や黄色（`#fbbf24`）で強調。
- **一貫したコンポーネント**: 共通のボタンクラス（`btn`, `btn-outline`）を使用。
- **列幅の最適化**: データの特性に合わせ、以下の列幅調整を行っています。
  - **製番**: 最小130pxを確保し、改行を防ぐ設定。
  - **担当者**: 漢字4文字を想定した最小幅を設定しつつ、長さに応じて可変。
  - **リンク**: コピーアイコンのみを表示するため、最小限の固定幅（80px）に制限。
  - **納期**: `yyyy-mm-dd` 形式が収まる固定幅（120px）を設定し、中央揃えで視認性を向上。
  - **各工程**: 内容に応じて個別に可変しつつ、最小100pxを確保。

## 2. 操作感の設計
- **モーダルベースの入力**: 複雑な設定は各工程のセルをクリックしてモーダルで集中して行う。
- **即時反映**: モーダルで「保存」を押した瞬間に一覧表が更新され、背後で同期が走る。
- **視覚的なフィードバック**: 
  - 保存成功時にトースト通知を表示。
  - 競合検知時にダイアログで状況を説明。

## 3. インタラクティブ要素とハイライト
- **連動ハイライト**:
  - 行内の**どこにマウスを乗せても、その案件の「製番情報エリア（左側8列）」がハイライト**されます。これにより、どの案件のデータを操作しているかを常に見失わないように設計されています。
  - **工程セルおよび完了ボタン**にマウスを乗せた時は、そのセル自体も個別にハイライトされ、クリック対象を明確に示します。
- **フィルタの使い分け**:
  - 右端の「完了を表示」ボタンは、過去案件の参照時のみONにします。
  - 日々の実務では「工程フィルタ」を活用し、「未手配」や「未出図」など、アクションが必要な案件だけを抽出して作業します。
- **ツールチップのレスポンス**: マウスの動きに合わせて滑らかに追従し、情報を遮らない位置に表示。
