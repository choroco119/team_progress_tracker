# 納入仕様書工程フィルタの強化 実装計画書

納入仕様書の進捗状況を詳細に絞り込めるよう、フィルタ機能を強化します。

## ユーザー要望
- 「工程未完フィルタ」を「工程フィルタ」に名称変更。
- 工程ごとにドロップダウンリストで詳細条件を選択可能にする。
- ドロップダウンの見た目（視認性）を担当者フィルタと統一する。

## 納入仕様書のフィルタ条件
1. **未出図**: 出図が済でない (`issueDate` が空)
2. **未返却**: 出図済だが、返却が済でない (`issueDate` があり、`returnDate` が空)
3. **修正要**: 修正要否が「要」 (`needsFix` === '要')
4. **未承認**: 出図または返却済だが、承認が済でない ((`issueDate` または `returnDate` があり) かつ `approvalDate` が空)

## 修正内容

### 1. index.html
- 「工程フィルタ」セクションのラベルと構造を変更。
- 納入仕様書用のセレクトボックスを `.filter-item` スタイルに変更して配置。

### 2. app.js
- フィルタ状態を管理する変数（`processFilterValues` 等）の確認。
- `renderTable` 内のフィルタリングロジックを上記の条件に基づいて更新。
- 「フィルタクリア」ボタンでドロップダウンもリセットされるように修正。
