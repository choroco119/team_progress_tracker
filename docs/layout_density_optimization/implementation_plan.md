# 表示密度の最適化（垂直方向の余白削減）

製番一覧の視認性を確保しつつ、一度に表示できる案件数を増やすため、全体の垂直方向の余白（パディング）を削減します。

## ユーザーレビューが必要な項目
- 余白の削減具合（今回は「少し削る」という方針に基づき、極端に詰めすぎない範囲で調整します）。

## 変更内容

### [CSS] [style.css](file:///c:/Users/kohei/.gemini/antigravity/scratch/team_progress_tracker/style.css)
以下の箇所のパディングを削減します。

1.  **ヘッダー**: 1.5rem → 1.0rem
2.  **カードヘッダー**: 1rem 1.5rem → 0.75rem 1.25rem
3.  **フィルタコントロール**: 1rem → 0.75rem
4.  **テーブルヘッダー (th)**: 0.75rem 1rem → 0.5rem 0.75rem
5.  **テーブルセル (td)**: 1rem → 0.6rem 0.75rem
6.  **タブボタン**: 0.75rem 1.5rem → 0.5rem 1.25rem

## 検証計画
### 手動検証
- ブラウザで `index.html` を開き、以下の点を確認します。
    - 1画面に表示される製番の行数が増えているか。
    - 各要素が窮屈になりすぎておらず、視認性が保たれているか。
    - ヘッダーやボタンの配置が崩れていないか。
